AWS Architecture Patterns

A reference guide for common cloud infrastructure patterns

Serverless Web Application

S3 → CloudFront → API Gateway → Lambda → DynamoDB

Static assets served from S3 via CloudFront CDN. API requests routed through API Gateway to Lambda functions. DynamoDB provides single-digit millisecond latency at any scale. No servers to manage.

Data Lake Pipeline

Data sources → S3 raw → Glue ETL → S3 processed → Athena

Raw data lands in S3. AWS Glue crawlers catalogue the schema automatically. Glue ETL jobs transform and clean the data. Athena enables SQL queries directly on S3 — no database server required. Pay only per query.

Real-time Streaming Pipeline

Producer → Kinesis → Lambda → DynamoDB

Events streamed into Kinesis Data Streams. Lambda consumers process each record in real time. Results stored in DynamoDB for low-latency reads. Handles millions of events per second with automatic scaling.

ML Model Deployment

S3 data → SageMaker train → Model registry → Endpoint → API Gateway

Training data stored in S3. SageMaker manages the full ML lifecycle — training, evaluation, and deployment. Models versioned in the model registry. Deployed as a real-time endpoint behind API Gateway for production use.

Security Best Practices

Least privilege IAM

Every service and user gets only the permissions they need. Use IAM roles, not access keys, for service-to-service communication.

Private S3 buckets

Never make S3 buckets public. Use CloudFront with Origin Access Control or pre-signed URLs for controlled access.

Encrypt everything

Enable S3 server-side encryption, DynamoDB encryption at rest, and enforce HTTPS on all endpoints.

CloudWatch monitoring

Set up alarms for Lambda errors, API Gateway 5xx rates, and unusual S3 access patterns.
